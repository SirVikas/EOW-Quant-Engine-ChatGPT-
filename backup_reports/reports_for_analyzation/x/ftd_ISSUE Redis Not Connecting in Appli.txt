**ISSUE: Redis Not Connecting in Application (TimeoutError)**

**Problem Summary:**

* Redis server is installed and running correctly on:

  * `127.0.0.1:6379`
* Manual validation successful:

  * `redis-cli ping → PONG`
* However, application logs show:

  * `Redis : not installed`
  * `[REDIS-HEALTH] Probe failed: TimeoutError`

---

**Root Cause:**

* Application is not using correct Redis configuration.
* Possible issues:

  * Hardcoded host (e.g., `"redis"` instead of `"127.0.0.1"`)
  * Missing/ignored `REDIS_URL`
  * Low/default timeout
  * Redis client not properly initialized

---

**Required Fix (MANDATORY):**

1. **Use Environment-Based Redis Connection**

```python
import os
import redis

def get_redis():
    url = os.getenv("REDIS_URL", "redis://127.0.0.1:6379")
    return redis.from_url(
        url,
        socket_connect_timeout=5,
        socket_timeout=5,
        decode_responses=True
    )
```

---

2. **Replace All Existing Redis Initializations**

❌ REMOVE:

```python
redis.Redis(host="redis", port=6379)
```

❌ REMOVE:

```python
redis.Redis(host="localhost", port=6379)
```

✅ REPLACE WITH:

```python
r = get_redis()
```

---

3. **Fix Redis Health Check**

```python
def check_redis():
    try:
        r = get_redis()
        r.ping()
        return True
    except Exception as e:
        print(f"[REDIS-ERROR] {e}")
        return False
```

---

4. **Set Environment Variable (Required)**

```powershell
setx REDIS_URL "redis://127.0.0.1:6379"
```

Restart terminal after setting.

---

5. **Ensure Dependency Installed**

```powershell
pip install redis
```

---

**Expected Result After Fix:**

* Logs should show:

  ```
  Redis : connected
  ```
* No more `TimeoutError`
* Engine should use Redis instead of in-memory cache

---

**Notes:**

* Do NOT rely on hardcoded hostnames
* Must support both local + production environments
* Timeout must be ≥ 5 seconds to avoid false failures

---

**Priority: HIGH**
This impacts caching, performance, and system stability.
