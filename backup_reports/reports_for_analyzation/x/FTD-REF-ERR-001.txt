# 📦 FTD-REF-ERR-001 — Global Variable Scope Error Fix (_last_skip)

---

## 🧠 1. CONTEXT

During engine startup, system crashes with:

```
SyntaxError: name '_last_skip' is assigned to before global declaration
```

📍 File: `main.py`
📍 Location: Around line ~283

---

## 🔍 2. ROOT CAUSE

Python scoping rule violation:

* `_last_skip` is being **assigned before declaring it as global**
* Inside a function, Python treats variables as **local by default**
* When `global _last_skip` appears AFTER assignment → conflict occurs

---

## ❌ CURRENT (INVALID PATTERN)

Example pattern found in code:

```python
_last_skip = something   # ❌ assignment happens

global _last_skip        # ❌ declared later (invalid)
```

OR:

```python
def some_function():
    if condition:
        _last_skip = 10   # ❌ local assignment first

    global _last_skip     # ❌ global declared later
```

---

## ⚠️ WHY THIS FAILS

Python compiler rule:

> If a variable is assigned anywhere in a function → it is treated as LOCAL
> Unless explicitly declared GLOBAL at the beginning

👉 Here:

* `_last_skip` becomes LOCAL first
* Then declared GLOBAL → ❌ conflict → SyntaxError

---

## ✅ 3. REQUIRED FIX (MANDATORY)

### 🔧 OPTION 1 — Minimal Fix (Immediate)

Move `global` declaration to the **TOP of the function**

---

### ✅ CORRECT PATTERN:

```python
def some_function():
    global _last_skip   # ✅ must be FIRST

    if condition:
        _last_skip = 10
```
